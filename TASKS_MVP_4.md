# TASKS_MVP_4.md

4차 MVP는 조용한 공존 기반 데스크탑 개입이 목표다.

## 31. 존재감 리듬 시스템

### 행동 밀도 제어

- [ ] 행동 과밀도 방지 구조 추가
- [ ] 장난 행동 최소 간격 추가
- [ ] 장시간 idle 유지 허용
- [ ] 연속 장난 방지
- [ ] 행동 사이 pause 추가
- [ ] 랜덤 딜레이 강화

### 사용자 반응 기반 조절

- [ ] 사용자가 무시하면 장난 빈도 감소
- [ ] 입력이 많은 시간에는 자동 얌전 모드 적용
- [ ] 조용히 같이 있는 시간 강화
- [ ] 존재감 과노출 방지
- [ ] 내부 energy 상태 검토
- [ ] 내부 socialTolerance 상태 검토
- [ ] sleep 기반 energy 회복 검토

## 32. 방해 강도 제어

### 기본 방해도

- [ ] 방해 강도 프리셋 추가
- [ ] 기본값은 낮은 방해 강도로 설정
- [ ] 강한 커서 방해 행동은 기본 비활성화
- [ ] 집중 모드 추가

### 사용자 보호

- [ ] 장시간 입력 중이면 장난 감소
- [ ] 전체 화면 앱 감지 시 조용해지기 검토
- [ ] 야간 시간대 행동 감소 검토
- [ ] 장시간 사용 시 에너지 감소 개념 검토

## 33. 약한 커서 관찰 행동

### 관찰 중심 반응

- [ ] 커서를 천천히 쳐다보기
- [ ] 커서 근처로 가끔 다가오기
- [ ] 커서 앞을 슬쩍 지나가기
- [ ] 발로 툭 치는 척하기
- [ ] 랜덤 blink 타이머 검토
- [ ] 정면/좌/우 에셋 매핑 검토

### 강한 방해 분리

- [ ] 지속적인 커서 추적 방지
- [ ] 클릭 방해 행동 옵션 분리
- [ ] 드래그 방해 행동 옵션 분리
- [ ] 커서 경로 막기 기본 비활성화

## 34. 최소 소품 시스템

### 소품 종류

- [ ] 발자국 소품 추가
- [ ] 털뭉치 소품 추가
- [ ] 작은 공 소품 추가
- [ ] 메모지 시스템 후순위 보류

### 소품 생명주기

- [ ] 소품 최대 개수 제한
- [ ] 일정 시간 후 자동 제거
- [ ] 사용자가 클릭하면 제거
- [ ] 소품 재생성 지연

## 35. 성격별 조용한 개입 차이

### 성격별 빈도

- [ ] 애교쟁이는 사용자 근처에 오래 머물기
- [ ] 새침이는 가까이 오면 살짝 피하기
- [ ] 겁쟁이는 구석과 안전거리 선호
- [ ] 느긋이는 장난 빈도를 낮게 유지
- [ ] 장난꾸러기는 짧고 드문 장난 강화
- [ ] 성격별 장난 빈도 상한 적용

## 36. 다중 모니터 구조 검토

### 구조 및 리스크

- [ ] 현재 모니터 작업 영역 기준 정리
- [ ] DPI 차이 리스크 기록
- [ ] 모니터 좌표계 리스크 기록
- [ ] 클릭 통과 경계 리스크 기록
- [ ] 실제 다중 모니터 이동 구현은 후속 MVP로 보류

## 37. 설정 재진입 UX

### 설정 진입 경로

- [ ] 트레이 메뉴에서 설정 열기 확정
- [ ] 고양이 우클릭 메뉴에서 설정 열기 검토
- [ ] 숨겨진 단축키 설정 열기 검토
- [ ] 평상시 화면 위 설정 버튼 완전 제거

### 설정 모드 동작

- [ ] 첫 실행 설정과 이후 설정 수정 흐름 분리
- [ ] 설정 모드에서만 입력 차단 허용
- [ ] 설정 모드에서만 리사이즈 UI 허용
- [ ] 설정 모드 종료 시 데스크탑 상호작용 상태 복귀

## 38. 4차 MVP 완료 조건

아래 조건을 만족하면 4차 MVP 완료로 판단한다.

- [ ] 고양이가 계속 행동하지 않는다
- [ ] idle 시간이 충분히 길다
- [ ] 장난 행동이 드물고 작게 발생한다
- [ ] 사용자가 작업 중이면 조용해진다
- [ ] 소품이 앱처럼 과하게 보이지 않는다
- [ ] 기본값에서 커서 방해가 피곤하지 않다
- [ ] 성격별 존재감 차이가 느껴진다
- [ ] 설정 진입이 데스크탑 위 상호작용 경험을 방해하지 않는다
- [ ] Desktop Goose류 감성이 기능보다 리듬으로 느껴진다
